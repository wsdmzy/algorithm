## 数组 链表 力扣题



## 分治
  一般适合递归实现
  - 每一层递归都会涉及这样三个操作
    1. 分解：将原问题分解成一系列子问题；
    2. 解决：递归地求解各个子问题，若子问题足够小，则直接求解；
    3. 合并：将子问题的结果合并成原问题。

  - 分治算法能解决的问题，一般需要满足下面这几个条件：
    1. 原问题与分解成的小问题具有相同的模式；
    2. 原问题分解成的子问题可以独立求解，子问题之间没有相关性(动态规划和问题规模有相关性)
    3. 具有分解终止条件，也就是说，当问题足够小时，可以直接求解；
    4. 可以将子问题合并成原问题，而这个合并操作的复杂度不能太高，否则就起不到减小算法总体复杂度的效果了。
